DLQ测试步骤

步骤1：创建测试数据

在表单中填写客户名称和金额，例如：

  • • 客户名称：DLQ Test Customer
  • • 金额：99.99
  • • Message ID：留空自动生成

步骤2：测试单个DLQ消息

点击 "Test DLQ" 按钮，创建一个会失败处理的消息

消息将被发送到死信队列，可以在DLQ标签页中查看

步骤3：测试多个DLQ消息

点击 "Multiple DLQ" 按钮，创建多个不同场景的失败消息

系统会自动创建3个不同金额的测试消息

步骤4：测试重试功能

切换到 "Dead Letter Queue" 标签页，可以：

  • • 点击 "Retry All" 重试所有消息
  • • 点击单个消息的刷新图标重试特定消息

步骤5：测试清理功能

在 "Dead Letter Queue" 标签页中：

  • • 点击 "Clear All" 清理所有死信消息
  • • 确认后所有数据将被清除

功能说明

Test DLQ 按钮

创建单个失败消息，发送到死信队列

Multiple DLQ 按钮

创建多个不同场景的失败消息

Retry All 按钮

重试死信队列中的所有消息

Clear All 按钮

清理死信队列中的所有消息

注意事项

确保应用程序正在运行

测试前检查统计页面的数据

Clear All操作不可撤销

使用内存数据库，重启后数据会丢失

控制台测试

在浏览器控制台中运行以下代码进行自动化测试：

// 加载测试脚本
const script = document.createElement('script');
script.src = '/dlq-test.js';
document.head.appendChild(script);

// 运行所有测试
testDLQ.all();

// 运行特定测试
testDLQ.single(); // 单个DLQ测试
testDLQ.multiple(); // 多个DLQ测试
testDLQ.retry(); // 重试功能测试
testDLQ.cleanup(); // 清理功能测试

预期结果

统计页面

  • • Dead Letter 计数会增加
  • • Total Processed 计数会增加
  • • Duplicates 计数应保持为0（除非同时测试重复检测）

死信队列页面

  • • 显示失败的消息详情
  • • 包含原始Message ID、客户名称、金额等信息
  • • 显示失败原因和失败时间

消息队列页面

  • • 显示处理失败的消息
  • • 状态显示为失败
  • • 包含失败原因信息
